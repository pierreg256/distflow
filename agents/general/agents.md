# Règles générales du projet

## Objectifs
- Garantir une base de code claire, testable et documentée.

## Règles de codage
- Utiliser TypeScript strict.
- Préférer des fonctions pures et des effets explicites.
- Nommer clairement les types, interfaces et fonctions.
- Limiter la portée des variables (const/let, éviter var).
- Documenter toute API publique.

## Règles de tests
- Ajouter des tests unitaires pour toute logique non triviale.
- Ajouter des tests d’intégration pour les flux critiques.
- Les tests doivent être déterministes et isolés.

## Règles de versioning
- Suivre SemVer pour les packages.

## Conventions Git
- Commits courts et descriptifs.
- Une fonctionnalité par branche.

## Qualité
- Pas de TODO non justifiés dans le code.
- Pas de code mort.
